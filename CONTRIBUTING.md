# Contributing

Thanks for contributing.

## What fits this repository

- Practical Harness Engineering guides
- Reusable `AGENTS.md` templates
- Playbooks for validation, contracts, observability, and microservices
- Small runnable demos that illustrate a specific Harness pattern

## What to include

- A clear problem statement
- A short explanation of why the material is useful
- Verification steps for demos
- Standalone writing: assume readers have not seen the original source repository

## Preferred style

- Short sections
- Concrete examples
- Practical sequencing
- Minimal jargon unless explained

## For demos

Each demo should include:

- `README.md`
- `AGENTS.md` when relevant
- Source files
- Tests or verification commands

## Pull request checklist

- The content is understandable without extra context
- The file is placed in the correct directory
- Any demo includes verification instructions
- The README or INDEX is updated if navigation changed
