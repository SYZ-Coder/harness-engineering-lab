# Service Overview

## Purpose

`order-query-service` provides read-oriented order views for UI and API consumers.

## What it exposes

- a query API for order summary lookup

## What it depends on

- `customer-service`
- `inventory-service`

## Why this metadata matters

This folder is intentionally small, but it models the minimum information a team or coding agent needs before changing a microservice:

- what the service is for
- who owns it
- which API it provides
- which dependencies might be affected by a change
